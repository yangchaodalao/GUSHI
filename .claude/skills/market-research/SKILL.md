# 市场深度研究 + 全日预测

**采集归采集，分析归分析。**

- `morning-brief` → 给你数据
- `market-research` → 给你判断
- `daily-market-news` → 给你验证

## 两种模式

| | 场景 1：全日预测 | 场景 2：深度研究 |
|---|---|---|
| **触发** | `预判今天` `分析今天走势` | `研究 XXX` `深究 XXX` `为什么 XXX` |
| **输入** | 早盘要闻数据 + 补充搜索 | 一个问题 |
| **输出** | `A股市场日报_YYYY-MM-DD全天预测.md` | `10_深度研究/主题.md` |
| **时机** | 8:30 拿到早报后 | 任何时候 |
| **频率** | 每个交易日一次 | 每天一个问题即可 |

## 场景 1：全日预测

### 信息来源

**必须读取**（不读不预测）：

1. 今日早盘要闻：`E:\IDEA\cc\Personal Cognitive OS\00_每日笔记\A股市场日报_{{TODAY}}早盘要闻.md`
2. 上一交易日收盘日报：`E:\IDEA\cc\Personal Cognitive OS\00_每日笔记\A股市场日报_{{LAST_TRADE_DATE}}.md`

**补充搜索**（按需，填补早报没有的信息）：

- A50 期货最新行情（如果早报中 A50 缺失）
- 今日重要经济数据发布时间表
- 隔夜美股板块 ETF 详细表现（XLK/XLE/XLF/XLV 等，如果早报没覆盖）
- 今日 A 股上市公司重要公告/限售解禁
- 今日新股申购

### 预测模板（7 模块）

参考 `assets/prediction_template.md`。必须包含：

```
① 开盘方向      → 验证节点：9:35
② 风格判断      → 验证节点：收盘
③ 板块强弱预测  → 验证节点：收盘
④ 主力资金方向  → 验证节点：收盘（北向 + 主力流入方向）
⑤ 情景推演      → 验证节点：收盘
⑥ 关键变量      → 验证节点：盘中
⑦ 验证节点表    → 全天
```

### 预测原则

**必须做：**

- 每个判断必须有数据支撑（"因为 A50 +0.8%，所以判断高开"）
- 用 `如果…则…` 而非 `一定会…`
- 给出概率权重（基准情景 ~60% / 不利情景 ~25% / 尾部 ~15%）
- 板块映射必须有逻辑链（"美股科技涨 → 情绪传导 → A股芯片/消费电子偏强"）

**禁止做：**

- ❌ 精确点位预测（"上证涨到 3450"）
- ❌ 单一方向断言（"今天必涨"）
- ❌ 没有数据支撑的直觉判断
- ❌ 只说利好不说利空

### 生成流程

1. 读取早盘要闻文件
2. 读取上一交易日收盘日报（了解前日市场状态：风格/情绪/主线）
3. 按需补充搜索（早报缺失的关键数据）
4. 逐模块填写预测
5. 直接写入 Obsidian（预测必须在开盘前落定，不等待确认）

## 场景 2：深度研究

### 研究方法：5 层逻辑树

研究一个问题时，不要只看表面。向下挖 5 层：

```
第一层：行业本身
  为什么国家扶持/限制？
  产业链结构是什么？
  核心驱动因素是什么？
  谁受益？谁受损？

第二层：国际政治
  中美/中歐/中日在这个领域的博弈是什么？
  为什么这个领域是核心战场？
  全球供应链如何分布？

第三层：中国国情
  为什么中国强调自主可控？
  国家为什么补贴？
  国企和民企在里面分别干什么？
  地方政府在其中扮演什么角色？

第四层：历史
  历史上有没有类似情况？
  日本/韩国/美国怎么走过这段路？
  哪些成功了？哪些失败了？为什么？

第五层：人性与资本
  为什么资金会追/会逃？
  为什么散户容易在高位接盘？
  为什么情绪周期会反复？
  市场的非理性是暂时的还是结构性的？
```

**不是每层都要填满。** 如果某个问题只挖到第 3 层就有完整答案，不用硬凑。关键是**向下挖**，不是**横向堆**。

### 研究原则

- 每天只研究一个问题
- 要有历史案例对照（没有历史参照的分析是空中楼阁）
- 要有反方观点（如果你只看到利好，主动找利空逻辑）
- 结论必须是**你自己的理解**，不是搜索结果的拼凑

### 搜索策略

分层搜索，由浅入深：

| 轮次 | 搜索内容 | 来源 |
|------|----------|------|
| 第 1 轮 | 现状 + 基本概念 | 百度百科/知乎/财联社 |
| 第 2 轮 | 政策逻辑 + 产业链 | site:gov.cn / 研报 |
| 第 3 轮 | 国际对比 + 历史案例 | 全网搜索 + CNBC/Reuters |
| 第 4 轮 | 反方观点 | "为什么 XXX 不会/不可能" |

## 共享行为

### 写入确认

- **场景 1（全日预测）**：生成后**直接写入 Obsidian**，不需要询问。预测必须在开盘前落定，不能等。
- **场景 2（深度研究）**：生成后**展示预览，询问用户是否写入**。研究笔记需要用户确认理解后再入库。

### 追问支持

同一对话内支持追问（"那 XX 呢？""和上次比呢？"）——不需要重新加载上下文。

### 自动推荐研究问题

当日收盘日报生成后，在报告末尾自动列出 2-3 个值得研究的问题：

```
💡 值得研究：
- [[芯片为什么持续强于大盘？]] — 政策驱动 vs 需求驱动
- [[北向资金连续流出意味着什么？]] — 历史规律
- [[地方财政困境如何影响银行股？]] — 传导机制
```

推荐规则：
- 基于当日市场最显著的特征（领涨/领跌板块、资金异动、政策信号）
- 问题要具体，不要泛泛（"研究一下股市" ❌ → "芯片板块为什么连续3天领涨？" ✅）
- 用 Obsidian 双链格式，方便直接创建

### 认知修正循环

收盘日报中如果发现了预测错误，在日报的"市场观察"里标注：

```
昨日预判回顾：
- ✗ 北向资金判断失误：预判流入，实际流出 20 亿
  可能原因：忽略了人民币贬值压力
- ✓ 芯片走强判断正确
```

这是整个系统最有长期价值的环节。
